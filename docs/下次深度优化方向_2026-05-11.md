## 🔴 深度反思 — 今天做得不够好的事

### 1. **过度迭代 → 缺少回归测试**

**症状**:每次新版本"沙箱测试通过"但用户在 Windows 上还是踩坑。
- v0.8 改了 `auth_required` 后用户进系统看到一堆 hidden,根本看不到账号管理
- v0.7 加 force_password_change 但前端没引导改密
- v0.6 加 ai_parse_itinerary 但前端 token 没刷新导致 401

**根因**:沙箱测的是**纯后端 API 行为**,真实用户走的是**前端整套链路**(浏览器 cookie / 缓存 / cors / 状态管理)。

**P0 修复**:
- [ ] 加 **Playwright E2E 测试** — 用真浏览器跑核心场景:登录 → 切 Tab → 创建报价 → 算价 → 导出
- [ ] 每个版本上线前必须跑通 E2E,失败不让 build
- [ ] 在 GitHub Actions 里跑 E2E

### 2. **编码 bug 反复出现 (LF/CRLF/BOM)**

**症状**:.bat 报错 → .ps1 中文乱码 → 推送脚本 URL 不展开
**根因**:Cowork 沙箱默认写 UTF-8 LF,Windows 期望 CRLF + (老 PS) GBK
**P0 修复**:
- [ ] 写一个 `tools/normalize_scripts.sh` — 自动把所有 .bat / .ps1 转 CRLF + 加 BOM
- [ ] CI hook: pre-commit 检查行结尾
- [ ] 文档里固定:**.bat 不放中文**(全英文,纯 ASCII)

### 3. **远程服务调用没有 dry-run**

**症状**:GitHub push 失败 4 次("Authentication failed" / "Repository not found" / "Permission denied")
**P0 修复**:
- [ ] 推送脚本前置:`Invoke-RestMethod /user` 验证 PAT 有效
- [ ] 推送脚本前置:`Invoke-RestMethod /repos/X/Y` 验证仓库存在 + 权限
- [ ] 失败时给 actionable 错误(而非通用 "Authentication failed")

### 4. **Docker rebuild 流程太重**

**症状**:每次小改前端 = 5-10 分钟重 build 镜像
**P0 修复**:
- [ ] **dev mode**:用 volume mount 把 `frontend/` 和 `backend/app/` 挂进容器,改文件实时生效(不用 rebuild)
- [ ] 加 `docker-compose.dev.yml` 专门跑开发模式
- [ ] 加 `--reload` 给 uvicorn(已支持)

### 5. **前端单文件已 4000+ 行,要拆**

**症状**:`components.js` 已经 4000+ 行,7 个组件全挤一起
**P1 修复**:
- [ ] 拆成 `components/QuoteBuilder.js` `components/SettingsPanel.js` 等
- [ ] 用 ES6 modules + `<script type="module">`
- [ ] 不引入 build 工具(保持单 HTML 部署优势)

### 6. **没有错误监控**

**症状**:用户报错全靠他截图;backend exception 只在 stdout
**P1 修复**:
- [ ] 接 Sentry 或自建 errorlog 表 (已有 UsageLog 加 error_msg)
- [ ] 前端 axios interceptor 把所有 4xx/5xx 写入 backend `/log/frontend-error`
- [ ] super_admin 的 admin 页加"错误日志"tab

### 7. **没有 Alembic, ALTER TABLE 补丁已 13 行**

**症状**:`database.py:additive_columns` 已经 13 行,继续加会乱套
**P1 修复**:
- [ ] 终于上 Alembic,把现有 schema 落成 baseline revision
- [ ] 删掉 `additive_columns` 整段
- [ ] 写一个 `scripts/migrate.sh` 一键 alembic upgrade head

http.interceptors.response.use(r => r, err => {
  if (err.response?.status >= 400) {
    httpRoot.post('/log/frontend-error', {
      url: err.config.url,
      status: err.response.status,
      message: err.response.data?.detail || err.message,
      stack: err.stack,
    }).catch(() => {});

  return Promise.reject(err);
});
