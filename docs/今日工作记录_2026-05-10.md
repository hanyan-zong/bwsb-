## 关键设计决策

### A. 赌自费策略架构(v0.5.2 业务规则)

**用户原话**(指引):
> 主结构只要有自由活动不能判定不赌自费, 主结构有自由活动剩下就是酒店使用级别 水上项目含的是否多 自由活动是否含餐 酒店是五星的少赌或者不赌 剩下就是亲子活动过多儿童过多 老年人超过55岁以上的过多少赌

**12 条 seed 策略覆盖业务规则**(优先级倒序):

| 优先级 | 条件 | 处理 |
|---|---|---|
| 100 | 无任何自由活动 | 不赌 |
| 95 | MICE/婚礼短行程 | 不赌 + 加 ¥100/人 |
| 90 | 老年(55+) ≥50% | 不赌 + 加 ¥80/人 |
| 85 | 5 星酒店+蜜月 | 让 ¥150/人 |
| 80 | 5 星酒店全团 | 让 ¥100/人 |
| 75 | 已含水上 ≥2 项 | 让 ¥80/人 |
| 70 | 餐+SPA+自由日含餐 | 让 ¥50/人 |
| 65 | 儿童占比 >50% | 让 ¥100/人 |
| 60 | 蜜月+旺季+多自由 | 让 ¥450/人 (主战场) |
| 50 | 年轻人+多自由 | 让 ¥350/人 |
| 45 | 亲子+自由 6~14h | 让 ¥250/人 |
| 1 | 兜底: 有自由活动 | 让 ¥200/人 |

### B. 5 角色权限分层(v0.7)

```
super_admin   公司超管 - 全权 + 跨 agency + 改设置 + 配额无限
ops_manager   公司 OP - 跨 agency 帮做单, 不改设置 - AI 月配额 1000
agency_owner  旅行社老板 - 本社全部 + 邀请下属 - AI 月配额 200
agent         业务员 - 仅自己 quote - AI 月配额 50
viewer        基础只读 - AI 配额 0
```

23 功能 × 5 角色 在 `feature_permissions.py:FEATURE_REQUIREMENTS` 集中声明。

### C. 配额双闸口(v0.7)

```python
require_feature(user, "ai_parse_itinerary")  # 角色 allowlist → 不在抛 403
consume_quota(db, user, "ai_parse_itinerary")  # 配额扣减 → 超额抛 429
```

**关键**:`consume_quota` 内部立即 `db.commit()` — 即使后续业务挂掉,配额也已扣 → 审计准确性硬要求。

### D. 自助注册三态机(v0.8)

```
status='pending_review'  → 自助注册成功后默认, login 返回 403 友好提示
status='active'          → 通过审核, 可登录
status='rejected'        → 拒绝, login 返回 review_note
status='disabled'        → 管理员停用
```

login 端点针对每种状态给**友好错误**(不是统一 401)。
