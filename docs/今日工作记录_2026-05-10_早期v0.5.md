## 第一段 · v0.4.1 部署修复（让项目真的能跑起来）

### A. 新增 Windows 一键启动脚本

| 文件 | 作用 |
|---|---|
| `一键启动.bat` | 双击执行 → 自动 cd /d "%~dp0" → 校验 docker-compose.yml → 检查 Docker → 自动建 .env → docker compose up -d --build → 轮询 health → 自动开浏览器 |
| `全盘搜索并启动.bat` | 用户不知道项目在哪儿时双击 → 先扫常见位置（OneDrive/Documents/Desktop）→ PowerShell `Get-ChildItem -Recurse` 全盘搜 C:\/D:\/E:\ → 找到后调用 `一键启动.bat` |

**踩坑**：用户实际目录是 `C:\Users\001\balijob\预报价系统B端版本`（**不是** Documents 子目录），OneDrive 重定向把"我的文档"映射成虚拟路径。所以一键启动 + 全盘搜两套方案。

### B. 前端 Vue DOM 错误修复

**症状**：登录卡片消失瞬间 Element Plus 抛 `Failed to execute 'insertBefore' on 'Node': The node before which the new node is to be inserted is not a child of this node.`

**根因**：`v-if` 频繁增删 DOM 节点，破坏 Element Plus el-tabs 内部 DOM ref 的引用。

**修复** (`frontend/index.html`)：
```html
<!-- 旧 -->
<div v-if="authRequired && !authenticated" class="login-wrap">...</div>
<el-tabs v-if="authenticated" ...>
<!-- 新 -->
<div v-show="authRequired && !authenticated" class="login-wrap">...</div>
<el-tabs v-show="authenticated || !authRequired" ...>
```

并把 `<script src="/static/js/app.js?v=2">` → `?v=3` 强制刷新缓存。

### C. 关闭口令门时白屏修复

**症状**：用户清空 `.env` 里 `BWS_AUTH_USERNAME` 后，访问 / 一片空白。

**根因**：`app.js` 里 `this.authenticated = r.data.authenticated`，关闭口令门时后端返回 `authenticated: false / auth_required: false` —— 前端死等"已登录"。

**修复** (`frontend/static/js/app.js`)：
```js
this.authenticated = r.data.authenticated || !r.data.auth_required;
// 同步 mounted() 里加 ↓
if (this.authenticated || !this.authRequired) {
  await this.loadAfterAuth();
}
```

### D. 部署文档新增踩坑记录

| 坑 | 怎么避 |
|---|---|
| 用户用 Python 3.14.4，pydantic-core/Pillow 没 cp314 wheel | 推荐装 Python 3.12 或直接走 Docker |
| Docker Hub 国内访问被墙 (`dial tcp 31.13.69.245:443: connectex...`) | Docker Desktop → Settings → Docker Engine 配 `registry-mirrors`: `https://docker.m.daocloud.io` / `dockerproxy.com` / `docker.1panel.live` |
| PowerShell 占位符乱码 | 命令模板里的占位符必须用 ASCII `<your-port>` 不能用中文"实际端口" |
| 用户把 docker 输出 `[+] Running 2/2` 当命令贴回 PowerShell | 文档说明白：那只是 docker 输出，无需重复执行 |

## 第二段 · v0.5 业务闭环（P0 三件套 + P1-7）

> 路线图来源：`docs/下次深度优化方向_2026-05-08.md`
> 完成度：P0-2 / P0-3 / P1-7 ✅； P0-1 (Alembic) 留下次（需要 docker 内 alembic init 配 env.py，复杂度独立）。

### A. P0-2 报价导出 PDF / Excel / Word ✅

新增 `backend/app/routers/exports.py`：

```
GET /api/v1/quotes/{id}/export?format=xlsx|pdf|docx
```

| 格式 | 工具 | 输出 |
|---|---|---|
| `xlsx` | `openpyxl` | 3 sheet：**封面汇总** / **逐日行程** / **价格明细** |
| `pdf` | `weasyprint` (HTML → PDF) | A4，BWS 品牌色 #2c5282，含中文 fallback (DroidSans + Noto CJK) |
| `docx` | `python-docx` | 复用母库雅加达瑟帕岛报价排版风格 |

**关键设计**：
- 复用 `filter_quote_dict(quote_dict, user)` —— **agent / viewer 导出文件自动隐藏 IDR 成本/利润/赌额**，与系统内权限策略一致
- 文件名 `BWS报价单_{quote_no}_{customer_name}.xlsx`
- HTTP 头 `Content-Disposition: attachment` + `Content-Type` 区分

**模板渲染**：jinja2 把 quote 字典塞进 HTML 模板（`templates/quote_pdf.html.j2`），WeasyPrint 直接吃。

### B. P0-3 赌自费回写闭环 + 策略胜率统计 ✅

新增 `routers/quotes.py` 端点：

```
POST /api/v1/quotes/{id}/feedback
Body: {
  "actual_optional_revenue_cny": 580,
  "actual_profit_cny": 1200,
  "won_or_lost": "won",  // won / lost / partial
  "notes": "客户买了水上+SPA，没买乌布"
}
```

写 `gamble_history` 表三个新字段：`actual_optional_revenue_cny` / `actual_profit_cny` / `won_or_lost` / `feedback_notes`（v0.4 已 ALTER TABLE 兼容）。

新增 `routers/settings.py` 端点：

```
GET /api/v1/settings/strategy-stats
→ [
    {
      "strategy_id": 7,
      "name": "巴厘 5 天 honeymoon 标准包",
      "hit_count": 12,
      "feedback_count": 8,
      "win_rate": 0.75,
      "avg_actual_profit_cny": 1850,
      "avg_recommended_cny": 450
    },
    ...
  ]
```

按角色：`super_admin` 看全社; `agency_owner` 看本社; `agent/viewer` 不可见（403）。

### C. P1-7 GambleResult 命中策略时仍带 OT 清单 ✅

`utils/gambling_engine.py`：fixed/per_pax 命中后，**继续填充** `configured_optional_tours` + `excluded_optional_tours`，方便业务员看见"我让的 ¥450 是希望客户买这几个自费"的清单。

### D. 前端集成

`frontend/static/js/components.js` QuoteHistory 组件新增：

2. **「✅ 团结束反馈」按钮** → 弹 modal：填实际收入 / 实际利润 / won/lost / 备注 → POST /quotes/{id}/feedback
